# SpoilGuard HTML Prototype Sample

This repository contains a sample zero-setup HTML prototype for the SpoilGuard project used in the course as an example for Semester Project Milestone 5: Rapid Prototyping & Draft Testing Plan.

## Purpose
This sample shows how a team can translate:
- problem definition
- requirements
- use cases
- user scenarios

into a simple interactive prototype.

## What This Prototype Demonstrates
The prototype includes sample flows for:
1. Scan Receipt (OCR simulation)
2. My Fridge / Inventory Management
3. Expiration Alerts
4. Savings / Waste Analytics
5. Accessibility Settings

## How to Run
No installation is required.

Option 1:
- Download `spoilguard.html`
- Double-click the file to open it in your browser

Option 2:
- Open the GitHub Pages link if published

## Files
- `spoilguard.html` — standalone interactive prototype
- `README.md` — project overview and usage notes
- `AI_promt.txt` — the prompt used to generate standalone interactive prototype
## Important Course Note
This repository is provided as a sample for instructional purposes.

Students may study:
- structure
- screen flow
- interaction logic
- organization of a sample prototype

Students may not copy:
- the SpoilGuard topic
- the wording of sample submission text
- the same screen labels and content without meaningful adaptation

Your own project submission must be based on your group’s unique problem, requirements, and use cases.

## Suggested Student Workflow
1. Review your Milestones 1–4
2. Map your 4 use cases to screen flows
3. Build a low- or mid-fidelity prototype
4. Demonstrate all required use cases
5. Prepare a testing plan for Milestone 6

## Accessibility Features Shown in This Sample
- high-contrast mode
- large font mode
- audio-alert indicator

## Disclaimer
This prototype simulates behavior for instructional purposes only. OCR, analytics, and notifications are mock interactions rather than production features.
